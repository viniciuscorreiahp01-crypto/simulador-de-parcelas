# Antigravity Agent Operational Rules

Before making any code changes the agent must:

1. Analyze the full project structure.
2. Read all relevant files before modifying them.
3. Understand the current implementation.
4. Avoid breaking existing functionality.
5. Preserve the architecture and code style.
6. Modify only the necessary files.

Development Protocol:

Before making changes read:

ANTIGRAVITY_RULES.md
package.json
vite.config.ts
src/main.tsx
src/App.tsx

Then inspect the existing implementation and apply the safest scalable solution.

Always explain the implementation plan before modifying files.
